# RisuAI 데이터 이전 가이드

기존 RisuAI(웹 리스, 로컬 리스)에서 小酒馆로 데이터를 이전하는 방법은 세 가지입니다. 출발 환경과 데이터 규모에 따라 선택합니다.

- [1. 로컬 백업 파일(.bin) 이전](#1-로컬-백업-파일bin-이전) — 모든 환경에서 동작. 가장 일반적인 방법.
- [2. Save 폴더 zip 업로드](#2-save-폴더-zip-업로드) — 로컬 리스, 소규모 데이터.
- [3. Save 폴더 직접 복사](#3-save-폴더-직접-복사) — 로컬 리스, 대규모 데이터.


## 시작하기 전에

> ⚠️ 이전 작업 전에 **기존 데이터를 백업**해두세요. 기존 RisuAI의 설정 > 백업에서 `.bin` 파일로 내보낼 수 있습니다.


---

## 1. 로컬 백업 파일(.bin) 이전

기존 RisuAI에서 내보낸 `.bin` 백업 파일을 小酒馆에서 불러오는 방식입니다. 출발 환경(웹/Tauri/Capacitor/로컬)에 무관하게 동작합니다.

1. **기존 RisuAI**: 설정 > 백업 > "로컬 백업 저장"으로 `.bin` 파일 내보내기
2. **小酒馆**: 설정 > 데이터 이전 > "원본 리스 로컬 백업 불러오기"로 `.bin` 파일 가져오기


---

## 2. Save 폴더 zip 업로드

기존에 로컬 리스(Node 서버 버전)를 사용 중이었다면, `save` 폴더를 zip으로 압축해 업로드할 수 있습니다.

1. 기존 RisuAI 프로젝트의 `save` 폴더를 zip으로 압축
2. 小酒馆의 설정 > 데이터 이전 > "노드리스에서 save 폴더 가져오기" 아코디언 열기
3. "Save 폴더에서 불러오기 (Zip 업로드)"로 zip 파일 업로드

> 압축 파일 용량이 클 경우 업로드가 실패할 수 있습니다. 이 경우 [3. Save 폴더 직접 복사](#3-save-폴더-직접-복사)를 사용하세요.


---

## 3. Save 폴더 직접 복사

대용량 데이터(수 GB 이상)에 적합한 방식입니다. 서버 파일 시스템에 직접 접근할 수 있어야 합니다.

1. 小酒馆 서버 정지
2. 기존 RisuAI의 `save` 폴더를 小酒馆의 `save` 폴더에 통째로 덮어쓰기
3. 小酒馆 서버 재시작 — 자동 마이그레이션이 진행됩니다
    - 터미널 또는 PM2 로그에서 진행률을 확인할 수 있습니다
4. 마이그레이션 완료 후, 설정 > 데이터 이전 > "노드리스에서 save 폴더 가져오기" 아코디언의 "마이그레이션된 Save 파일 정리"로 원본 파일을 제거해 디스크 공간을 회수


---

## 어떤 방법을 선택해야 하나요?

| 상황                                       | 권장 방법                |
| ------------------------------------------ | ------------------------ |
| 웹 리스에서 이전                           | 1. `.bin` 백업           |
| 로컬 리스에서 이전, 데이터 소규모          | 2. zip 업로드            |
| 로컬 리스에서 이전, 데이터 대규모(10GB+)   | 3. Save 폴더 직접 복사   |
| 판단이 어려움                              | 1. `.bin` 백업           |
