## MySQL 表结构

| 表 | 用途 | 关键字段 |
|---|---|---|
| `users` | 用户账号 | `id`, `username`, `password_hash` |
| `characters` | 角色卡 | `id`, `user_id`, `name`, `card_data`(JSON), `tags`(JSON) |
| `user_settings` | 用户设置 | `user_id`, `preferences`(JSON), `presets`(JSON), `plugins`(JSON), `modules`(JSON) |
| `character_assets` | 角色资源 | `id`, `user_id`, `character_id`, `asset_type`, `data`(BLOB) |

## API 路由（新增）

| 方法 | 路径 | 认证 | 说明 |
|---|---|---|---|
| GET | `/api/auth/config` | 无 | 返回 auth 模式 |
| POST | `/api/auth/register` | 限流 | 注册 |
| POST | `/api/auth/login` | 限流 | 登录 |
| POST | `/api/auth/logout` | JWT | 注销 |
| GET | `/api/user/characters` | JWT | 角色列表 |
| GET | `/api/user/characters/:id` | JWT | 角色详情 |
| PUT | `/api/user/characters/:id` | JWT | 创建/更新角色 |
| DELETE | `/api/user/characters/:id` | JWT | 软删除角色 |
| GET | `/api/user/settings` | JWT | 读取设置 |
| PUT | `/api/user/settings` | JWT | 保存设置 |

## 关键代码路径

```
前端渲染链 (App.svelte):
  aprilFools → auth 登录页 → Loading → Settings → Mobile → 主布局

认证流程:
  bootstrap.ts → getAuthConfig() → 等待登录 → injectToken() → 加载数据

数据流:
  登录成功 → JWT 存 localStorage → NodeStorage.injectToken()
  API 调用 → risu-auth header → server checkAuth → req.user
```

## 开发注意事项

### 1. 不要存储敏感数据到服务器
以下字段在保存时需从数据库对象剥离（**Phase 3 待实现**）：
`openAIKey`, `claudeAPIKey`, `proxyKey`, `NAIApiKey`, `elevenLabKey`, `voyageApiKey`, `openrouterKey`, `nanogptKey`, `huggingfaceKey`, `fishSpeechKey`, `cohereAPIKey`, `stabilityKey`, `falToken`, `vertexPrivateKey`, `vertexAccessToken`, `OaiCompAPIKeys`, `deeplOptions`, `deeplXOptions`, `novelai.token`, `mancerHeader`, `google.*`, `account.*`

### 2. 聊天内容不存服务器
当前聊天数据存在 IndexedDB，不应上传到 MySQL。如需跨设备同步聊天，需另外设计端到端加密方案。

### 3. i18n 规范
- 新增 UI 文本 → 先加到 `cn.extra.ts` 的对应模块
- 翻译 key 按功能分组：`auth.*`, `realm.*`, `downloadProcess.*`, `tos.*`
- 英文 fallback 加到 `en.ts`
- 语言合并顺序：`merge(English, cn, cn.extra)` → 中文优先

### 4. 后端修改注意事项
- `server.cjs` 是 6500+ 行的单体文件，新增路由加在相关区域
- 认证路由在 `loginRouteLimiter` 区域附近
- 所有数据 API 需要 `checkAuth(req, res)` 检查
- `checkAuth` 现在会自动设置 `req.user`
- 关闭服务器前会调用 `closePool()` 清理 MySQL 连接

### 5. 前端修改注意事项
- 使用 Svelte 5 runes（`$state`, `$derived`, `$effect`）
- UI 组件优先用 Sh* 系列（`ShButton`, `ShInput`, `ShDialog` 等）
- 路由不是 URL-based，是 store-based（`settingsOpen`, `selectedCharID` 等）
- App.svelte 是顶层渲染路由器，修改视图层级时需注意条件顺序
